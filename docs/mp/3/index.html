    密评“应、宜、可”极简版

    2023-07-11
    2023-07-12
    2 min read
    GB
    mp
    Hits

      本文是博主在同事对密评相关国标和行标归纳总结的基础上，又提炼出的“极简版”——只保留了各个控制点的“应、宜、可”，而不关注具体的条款内容。

    物理和环境安全

    控制点第四级第三级第二级第一级
    身份鉴别应宜宜可
    电子门禁存储完整性应宜可可
    视频监控存储完整性应宜//

    网络和通信安全

    控制点第四级第三级第二级第一级
    身份鉴别应应宜可
    通信数据完整性应宜可可
    通信数据机密性应应宜可
    网络边界访问控制信息完整性应宜可可
    安全接入认证宜可//

    设备和计算安全

    控制点第四级第三级第二级第一级
    身份鉴别应应宜可
    远程管理通道安全应应//
    系统资源访问控制完整性应宜可可
    重要信息资源安全标记完整性应宜//
    日志记录完整性应宜可可
    重要可执行程序完整性和来源真实性应宜//

    应用和数据安全

    控制点第四级第三级第二级第一级
    身份鉴别应应宜可
    访问控制信息完整性应宜可可
    重要信息资源安全标记完整性应宜//
    重要数据传输机密性应应宜可
    重要数据存储机密性应应宜可
    重要数据传输完整性应宜宜可
    重要数据存储完整性应宜宜可
    不可否认性应宜宜可

    管理制度

    控制点第四级第三级第二级第一级
    具备密码应用安全管理制度应应应应
    密钥管理规则应应应应
    建立操作规程应应应/
    定期修订安全管理制度应应//
    明确管理制度发布流程应应//
    制度执行过程记录留存应应//

    人员管理

    控制点第四级第三级第二级第一级
    遵守法律法规和管理制度应应应应
    建立密码应用岗位责任制应应应应
    建立上岗人员培训制度应应应/
    定期安全岗位人员考核应应应/
    建立关键岗位人员保密和调离制度应应//

    建设运行

    控制点第四级第三级第二级第一级
    制定密码应用方案应应应应
    制定密钥安全管理策略应应应应
    制定实施方案应应应应
    投入运行前进行密评应应宜可
    定期开展密评及攻防演习应应//

    应急处置

    控制点第四级第三级第二级第一级
    应急策略应应应可
    事件处置应应//
    向有关部门上报应应//
