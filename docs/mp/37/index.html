    密评对代码审查能力的要求

              密评标准体系图

    从《密评测评实施指引》看

    1. 测评方式
      1. 第一类：以说和看为主，例如访谈、文档审查、实地查看
      2. 第二类：以实操为主，例如工具测试、配置检查、代码审查、试错测试
      3. 第三类：以验证为主，例如
        1. 跟踪测试：对实际或模拟的密码保护过程进行完整跟踪，确认安全机制、调用方法/频次的正确性
        2. 基于密码机制的主动分析：篡改、重放、中间人攻击等

      获取证据的可靠程度为：“第一类” < “第二类”或“第三类”。在所提供的第二类测评方式中，“配置检查”更适用于对合规密码产品使用；“代码审查”更适用于对开源代码库等使用，而且需要注意核实被测方所提供的静态代码与业务系统实际运行代码的一致性，例如版本信息。

      第一类测评方式属于所有测评指标默认采用的方式，且不局限于现场测评使用，后文不做赘述。

    密码误用 1：与密码算法安全强度相关

      规则：不得使用安全强度不足的密码算法

    1. 密码算法安全强度：指破解密码算法所需要的计算量大小（单位是比特 bit）
      1. n 比特安全强度，即攻破该密码算法需要 $2^n$ 次计算
      2. 类比对称密码算法的穷举攻击所需要的工作量
    2. 被认为是“安全的”密码算法的安全强度是动态变化的，与攻击手段、计算能力的提升相关，NIST SP 800-57（Recommendation for Key Management）……

      对称密码算法和公钥密码算法的安全强度

    安全强度（比特）对称密码算法基于有限域离散对数（如 DSA，DH，MQV）L 为公钥比特长度，N 为私钥比特长度基于大整数分解（RSA）k 为公钥比特长度基于椭圆曲线离散对数（如 ECDSA，EdDSA，ECDH，ECMQV 等）f 为私钥比特长度

      分为两种情况：

    1. 数字签名算法等依赖抗碰撞的应用：需要依赖抗碰撞性（强/弱抗碰撞性）
    2. HMAC，KMAC，KDF和随机数生成等：只需要保证 pre-image resistance 抗（第二）原像攻击（即弱抗碰撞性），不需要依赖抗碰撞的应用
